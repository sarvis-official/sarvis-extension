# Contributing to Sarvis

Thank you for your interest in contributing to Sarvis.

We welcome contributions including bug fixes, improvements, documentation updates, feature suggestions, and performance optimizations.

## Getting Started

### 1. Fork the Repository

Fork the repository and clone it locally.

```bash
git clone https://github.com/sarvis-official/sarvis-extension.git


2. Install Dependencies
npm install
3. Start Development
npm run watch

Press F5 in VS Code to launch the Extension Development Host.

Project Structure
src/        -> Extension source code
media/      -> Icons and UI assets
dist/       -> Compiled files
Contribution Guidelines
Follow clean code principles
Keep code modular and reusable
Use TypeScript best practices
Maintain consistent formatting
Avoid unnecessary dependencies
Ensure all builds pass before submitting PRs
Pull Request Process
Create a new branch
git checkout -b feature/your-feature-name
Commit changes clearly
git commit -m "Add feature"
Push branch
git push origin feature/your-feature-name
Open a Pull Request
Reporting Issues

When reporting issues include:

VS Code version
Extension version
Operating system
Error logs/screenshots
Steps to reproduce
Feature Requests

Feature suggestions are welcome through GitHub Issues.
